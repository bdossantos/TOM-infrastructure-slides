Description du projet

Refonte de l'infrastructure de la société The Optin Machine


  • Base de Collecte
  • Supports de Collecte
  • Tracking
  • Serveurs "statiques"

L'existant

  • 1 serveur d'application (Apache, PHP ...)
    • Jeux, Overlays ...
    • Tracking
  • Base de données MySQL :
    • 1 Maître pour l'écriture
    • 1 Esclave pour la lecture
  • Serveurs statiques
    • 2 Lighttpd derrière un Round Robin pour l'équilibrage de charge

L'existant

https://cacoo.com/diagrams/dND4SXZLkcGWQnaI

Limites / Inconvénients

La Haute Disponibilité :


  • SPOF au niveau du Front
  • Pas de "Failover" automatique pour la DB
    • Passage de l'esclave en Maître en cas de défaillance du maître

Conception de la base de données :


  • Champs incorrectement typé
  • Champs non indexé
  • Mauvais choix du moteur (MyISAM au lieux d'InnoDB)
    • Tables bloquées en écriture lors de SELECT ...

Limites / Inconvénients

Opération sur la base de données :


  • Fastidieux à
    • Répliquer : 70 GO de données, besoin de "lock" le maître ...
    • Sauvegarder
      • Utilisation de MySQL Dump
      • Impossibilité de faire des "Snapshots" des volumes

Limites / Inconvénients

Tracking :


Application Node.js (v2.x à cette époque) + écriture de log au format text

  • Instabilité du service (crash/redémarrage sans raison du serveur)
  • Fuites de mémoire
  • Soucis d'I/O
  • Pas de haute disponibilité, non distribué

Objectifs de la refonte

  • Améliorer la disponibilité de l'infrastructure
  • Régler les problèmes liés à la base de données
  • Refactoring complet du tracking
  • Automatiser au maximum les opérations et déploiement de l'infrastructure
    • Gestion des configurations
    • "Orchestration" des serveurs
    • "Infrastructure as code"

      # Vagrantfile

        config.vm.box = "squeeze64"
        config.vm.box_url = "https://domain.tld/vagrantboxes/squeeze64.box"

        config.vm.provision :puppet do |puppet|
          puppet.manifests_path = "manifests"

        end

        config.vm.define :couchdb do |couchdb_config|
          couchdb_config.vm.host_name = "couchdb"
          couchdb_config.vm.network :hostonly, "192.168.33.10"
          couchdb_config.vm.forward_port 5984, 5984
          couchdb_config.vm.forward_port 9200, 9200
        end
      end

      def puppet_apply(module='', args='', options='--modulepath /etc/puppet/modules'):
          """
          Apply Puppet module

          eg : fab -R db puppet_apply:module=couchdb
          """
          if module != '':
              print(green("Apply '%s' Puppet module to %s" % (module, env.host_string)))
              options += ' --logdest console --logdest syslog'
              run("puppet apply -e 'include %(module)s %(args)s' %(options)s" % locals())
          else:
              print(red('Module name required !'))
